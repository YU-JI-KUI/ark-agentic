# 句级流式输出频率控制方案

## 背景

当前每个 LLM token chunk 直接经由 `on_content_delta` → `StreamEventBus._emit()` → Queue → SSE 输出，导致客户端每秒收到数十个细碎事件。目标是将多个 token 聚合为完整句子后再发出。

## 关键链路位置

```
runner._scoped_content(delta)
    → SentenceBatchingHandler.on_content_delta(delta)   ← 新增拦截层
        → SentenceBuffer.feed(delta) → [完整句子列表]
            → StreamEventBus.on_content_delta(sentence)  ← 现有代码不变
```

## 需要新增/修改的文件

- **新增** `[src/ark_agentic/core/stream/sentence_buffer.py](src/ark_agentic/core/stream/sentence_buffer.py)` — 纯工具类，持有 `_buf: str`，提供 `feed(delta) -> list[str]` 和 `flush() -> str | None`
- **新增** `[src/ark_agentic/core/stream/batching_handler.py](src/ark_agentic/core/stream/batching_handler.py)` — 实现 `AgentEventHandler` 协议的装饰器，包装任意 inner handler，拦截 `on_content_delta`，其余方法透传
- **修改** `[src/ark_agentic/api/chat.py](src/ark_agentic/api/chat.py)` — 在构建 `StreamEventBus` 后，根据 feature flag（如请求参数 `sentence_mode=true` 或配置项）决定是否套一层 `SentenceBatchingHandler`
- **修改** `[src/ark_agentic/core/stream/__init__.py](src/ark_agentic/core/stream/__init__.py)` — 导出 `SentenceBuffer` 和 `SentenceBatchingHandler`

## 句边界检测逻辑

```python
# 中文句末：。！？ 及其叠用（……）
# 英文句末：. ! ? 后必须跟空白（避免 e.g. / 1.5 误判）
# 段落分隔：连续两个以上换行
SENTENCE_BOUNDARY = re.compile(r'[。！？…]+|(?<=[.!?])\s+|\n{2,}')
```

## flush 触发时机

`SentenceBatchingHandler` 在以下情况强制 flush 缓冲区剩余内容：

- `emit_completed()` 调用前（`StreamEventBus` 已有此方法，在 `_close_text_message` 前插入）
- `emit_failed()` 调用前（同上）
- 建议：在 `SentenceBatchingHandler` 上额外暴露 `flush()` 方法，由 `chat.py` 在 `run_agent()` 结束后显式调用

## 开关设计

在 `chat.py` 的请求处理逻辑中：

```python
handler: AgentEventHandler = bus
if settings.sentence_mode or request.params.get("sentence_mode"):
    handler = SentenceBatchingHandler(inner=bus)
```

不影响任何现有调用路径，完全向后兼容。
