# CLI Init Enhancement Design

Date: 2026-03-13

## Summary
Improve `ark-agentic init` to be PA-first by default, align docs with actual CLI options, and scaffold a more complete starter project without generating a real `.env`. Move the CLI-only `index.html` into the CLI package, add README quickstart, `.gitignore`, and a minimal tool + skill example so the project runs out-of-the-box.

## Goals
- PA-first defaults (`--llm-provider=pa-sx`).
- `.env-sample` remains the only generated env file.
- Generated README contains end-to-end quickstart for CLI/API/Studio flows.
- Generated project includes a minimal tool + skill example wired into the default agent.
- Add `.gitignore` with common patterns.
- Keep existing CLI flags compatible.
- Relocate the CLI scaffold `index.html` into `src/ark_agentic/cli/`.

## Non-Goals
- Do not generate `.env` with secrets.
- Do not add new CLI commands.
- Do not refactor core LLM env parsing.

## Current State
- CLI is in `src/ark_agentic/cli/main.py` and templates in `src/ark_agentic/cli/templates.py`.
- `init` generates `pyproject.toml`, `main.py`, default agent skeleton, optional API/Studio, and `.env-sample`.
- `.env-sample` is generated by `_render_env_sample` with provider-specific blocks.
- CLI copies `src/ark_agentic/static/index.html` into generated projects when `--api` is enabled.

- Change default `--llm-provider` to `pa-sx`.
- Update init output text to include PA-first guidance and `.env-sample` -> `.env` manual step.
- If `index.html` cannot be copied, print an explicit note.
- Source `index.html` from the CLI package path (not `ark_agentic/static`).
- Change default `--llm-provider` to `pa-sx`.
- Update init output text to include PA-first guidance and `.env-sample` -> `.env` manual step.

### 2) CLI HTML Asset Location
- Move `src/ark_agentic/static/index.html` to `src/ark_agentic/cli/index.html`.
- Update CLI copy logic to read from `Path(__file__).parent / "index.html"`.
- Update API template in `templates.py` to reference the new CLI path when scaffolding.
- Update tests that assert the static index location to target the new path.
- Keep `ark_agentic/static` for app-level assets (`insurance.html`, `securities.html`, etc.).

Add new templates to `src/ark_agentic/cli/templates.py`:
- `README_TEMPLATE`: project-level quickstart with separate instructions for:

  - Studio mode (`ENABLE_STUDIO=true` + `AGENTS_ROOT=...`)
  - PA-JT key explanations
- `GITIGNORE_TEMPLATE`: env, __pycache__, data, dist/build, .venv, IDE files.
- `EXAMPLE_TOOL_TEMPLATE`: minimal `tools/echo.py`.
- `EXAMPLE_SKILL_TEMPLATE`: `skills/echo/SKILL.md` with a single instruction.

## User Experience Flow
1) User runs `ark-agentic init foo`.
2) CLI reads `src/ark_agentic/cli/index.html` and scaffolds it into the generated project when `--api` is enabled.
3) Scaffold includes README, .gitignore, example tool + skill.
4) User copies `.env-sample` to `.env` manually and fills required keys.
5) User runs CLI or API; example tool can be invoked immediately.

## Acceptance Criteria
- `ark-agentic init foo` creates README, .gitignore, example tool/skill.
- Default provider is PA-SX.
- README instructions are accurate for CLI/API/Studio.
- Example tool is registered and usable without extra edits.
- No `.env` is generated.
- CLI uses `src/ark_agentic/cli/index.html` as the scaffold source.

## Rollout
- Update CLI templates and init logic.
- Move the CLI-only `index.html` into `src/ark_agentic/cli/` and adjust tests accordingly.
- Update README in the root repo to align with provider choices (if requested later).

## Risks
- Changing default provider may surprise external users; README must clarify optional OpenAI path.
- Moving `index.html` requires updating tests and any direct path references.
