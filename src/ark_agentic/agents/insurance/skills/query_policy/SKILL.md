# 保单查询技能

> **输出原则（最高优先级）**
>
> 用户在移动端查看：A2UI 卡片已占据主屏幕，卡片内已完整呈现每张保单的名称、保单号、年度、各项金额。
> 卡片后再追加文字描述会打断阅读体验。
>
> 因此：`render_card` 调用后，**只允许 1 句文字（≤ 25 字）**作为引导，其余一律省略。
> 严禁在文字中重复保单名称、金额、保单号等任何卡片已展示的内容。

## 触发条件

满足以下**任一情形**时触发：

- 用户询问自己有多少保单、有哪些保单、保单列表
- 用户询问持有哪些保险产品、保险概况
- 用户询问某张保单的可用额度明细（生存金、红利、贷款、退保金）

**不触发**的情况：

- 用户表达取款/领钱/借款等执行意图 → 由 `withdraw_money` 技能处理
- 用户询问"能领多少钱"、"最多能取多少" → 由 `withdraw_money` 技能处理（容量查询）
- 用户说"想取钱"但没说金额 → 由 `clarify_need` 技能处理

## 执行流程

### 第一步：获取保单数据

调用 `rule_engine` 获取用户全部保单信息：

```
rule_engine(action="list_options", user_id=用户ID)
```

不传 `amount`，返回所有保单的完整可用额度。

### 第二步：渲染保单卡片

调用 `render_card` 展示保单详情列表卡片：

```
render_card(card_type="policy_detail")
```

`card_args` 可选，所有字段均可不传（提取器有合理默认值）：

```json
{
  "page_title": "您的保单详情",
  "prompt_text": "如需了解某张保单详情，请告诉我保单名称。"
}
```

### 第三步：引导文字

卡片已完整展示所有保单信息。**禁止**在文字中重复保单名称、金额、保单号等任何卡片内容。

仅允许 1 句引导（≤25字），示例：
- "以上是您的保单概况，有什么想了解的吗？"
- "如需办理取款，请告诉我金额。"

## 输出格式

**主呈现 = A2UI 保单详情卡片（`policy_detail`）**。卡片渲染成功后，文字部分**严格限制**在 1 句以内（≤25字）。

## 风格要求

- 友好、专业、简洁
- 不堆砌术语
- 金额使用千分位格式（如 65,000 元）
