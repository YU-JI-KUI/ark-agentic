# 方案改写技能

当用户对推荐的方案提出修改意见时，使用此技能调整方案。

## 触发条件

用户表达以下意图时触发：
- "金额能多一点吗"
- "有没有更快到账的方案"
- "能不能不要影响保障"
- "换一个方案"
- "我觉得利息太高了"

## 改写流程

### 1. 理解修改意图

分析用户的反馈，识别需要调整的维度：
- 金额调整
- 时间要求调整
- 费用/利息敏感
- 保障影响敏感
- 完全更换方案

### 2. 检查可行性

调用工具验证修改是否可行：

```
rule_engine(action="calculate_xxx", amount=新金额, ...)
```

### 3. 生成新方案

如果修改可行，生成调整后的方案：
- 明确说明做了哪些调整
- 对比原方案的差异
- 说明新方案的优缺点

如果修改不可行，说明原因并提供替代建议。

## 常见修改场景

### 金额调整

用户："能不能多取一点？"

响应示例：
```
好的，我来看看能否调整金额。

经查询，您的保单最多可操作：
- 部分领取：最高65,000元（已是上限）
- 保单贷款：最高33,600元

如果您需要更多资金，可以考虑：
1. 同时使用两种方式，合计可取约98,600元
2. 如果仍不够，可能需要考虑退保（但会有损失）

您看这样可以吗？
```

### 时间要求调整

用户："能快点到账吗？"

响应示例：
```
理解，您需要更快到账。

各方式到账时间：
- 保单贷款：1-2个工作日 ✅ 最快
- 部分领取：3-5个工作日
- 退保：5-7个工作日

建议选择保单贷款，可最快1个工作日到账。
您看这个方案可以吗？
```

### 费用敏感

用户："利息能少点吗？"

响应示例：
```
我理解您关注利息成本。

保单贷款的利率是固定的（年利率5.5%），无法调整。

但如果您只是短期周转：
- 如果1个月内还款，利息约150元
- 如果3个月内还款，利息约460元

或者您可以选择部分领取，完全没有利息费用。
您更倾向哪种方式呢？
```

## 注意事项

1. 始终尊重用户的偏好
2. 提供不可行时的替代方案
3. 清晰对比调整前后的差异
4. 确认用户对新方案的接受度
