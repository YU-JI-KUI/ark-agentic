# 需求澄清技能

当用户表达了取钱意图但**尚未说明金额**时，用本技能做最少必要询问后进入取款方案。

## 原则

- **必问项只有金额** — 无金额无法算方案，必须问清。
- **用途、到账时间均为可选** — 可简短问一句，用户不愿说则直接进入方案，不追问。
- **不问已知信息** — 用户已提到的内容一律不再问。
- **问完即推进** — 不寒暄、不重复确认，信息够用就交给 withdraw_money。

## 触发与不触发

- **触发**：用户表达**执行意图**（「想取钱」「需要用钱」「帮我取一些」等），且**未给出金额**。
- **不触发**（以下情况直接走 withdraw_money）：
  - 用户已给出金额（如「取5万」「需要10万」）→ 直接展示取款方案卡。
  - 用户询问可领额度上限（如「我能领多少钱」「最多能取多少」「有多少额度」「能取多少」）→ 这是**容量查询**，非执行意图，由 withdraw_money 展示汇总卡，无需询问金额。

## 执行步骤

### 1. 询问金额（必做）

用户未说金额时，只问一句：

- 示例：「请问您大概需要多少资金？」

不展开原因、不给选项，一句即可。

### 2. 可选补充（一句带过）

金额确认后，若希望优化推荐，可**只问一句**把用途和到账时间都带上：

- 示例：「方便简单说下用途或到账时间吗？不说也可以直接为您算方案。」

用户不答或不愿说 → 立即进入取款流程，不再问。

### 3. 交接

金额已有（用途/时间有无均可）→ 本技能结束，由 withdraw_money 出方案。

## 注意

- 不重复问用户已说的金额、用途、紧急程度。
- 不问「从哪张保单取」— 由规则引擎自动匹配。
- 澄清阶段不给出具体可领/可贷金额，金额以规则引擎计算结果为准。
