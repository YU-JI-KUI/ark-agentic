# 需求澄清技能（合规标准流程）

当用户表达了取钱意图但**尚未明确金额**时，执行此标准化采集流程。

> ⚠️ 本流程包含监管要求的标准提问步骤，不可省略或替代。

## 核心原则

1. **金额是必须的信息** — 没有金额无法进入方案计算，必须获取
2. **用途是标准采集项** — 主动询问，帮助推荐更合适的方案
3. **紧急程度是标准采集项** — 主动询问，影响方案排序优先级
4. **信息齐全后立即推进** — 不要啰嗦，直接进入取款流程（withdraw_money 技能）

## 不问已知信息

在提问之前，必须先检查用户已经说了什么。以下信息如果用户已经提供，**绝对不要重复询问**：

| 信息项 | 用户可能的表述 | 已知则跳过 |
|-------|-------------|-----------|
| 金额 | "取5万"、"需要10万"、"3万块" | 不再问金额 |
| 用途 | "交学费"、"看病"、"急用" | 不再问用途 |
| 紧急程度 | "很急"、"不着急"、"越快越好" | 不再问时间 |

**只问用户没有提到的信息。如果金额已知，直接进入取款流程，不要多此一举。**

## 触发条件

以下情况触发本技能：
- 用户说"我想取钱"、"需要用钱"等，但**未说明金额**

以下情况**不触发**本技能：
- 用户已经明确了金额（如"我想取5万"）→ 直接进入 withdraw_money 技能
- 用户在聊保单查询、保障咨询等非取款话题

## 执行流程

### 第一步：询问金额（必须）

用户说了"想取钱"但没说金额时，简短询问：

```
请问您大概需要多少资金呢？
```

不要给选项列表，不要解释为什么需要这个信息，一句话即可。

### 第二步：采集用途和紧急程度（标准步骤）

金额确认后，如果用途和紧急程度未知，**主动询问**：

```
方便告诉我这笔资金的用途吗？比如是教育、医疗还是其他？另外，时间上大概什么时候需要到账？这样我能为您推荐最合适的方案。
```

**注意**：
- 这是标准采集步骤，应主动询问（而非等用户自己提及）
- 如果用户明确不想回答，尊重其意愿，直接用已有信息进入下一步
- 不要反复追问，一次未答即跳过

### 第三步：交接给取款流程

信息采集完成后（至少金额已知），本技能结束，交接给 withdraw_money 技能进行方案计算。

## 注意事项

1. **最重要**：不要重复询问用户已经明确的信息
2. **不要啰嗦**：信息够了就推进，不要多余的确认和寒暄
3. **不需要保单信息**：不要问用户"从哪份保单取"，规则引擎会自动处理
4. 不要让用户觉得在被审问
5. 尊重用户隐私，如果用户不愿回答可选信息，直接用已有信息进入方案计算
6. 不要在澄清中提及具体金额数字（如"可贷款3.3万"），金额需通过工具计算后才能给出
