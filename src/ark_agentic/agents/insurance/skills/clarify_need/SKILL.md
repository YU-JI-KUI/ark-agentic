# 需求澄清技能

当用户表达了取钱意图但**尚未明确金额**时，使用此技能通过简短的对话收集金额信息。

## 核心原则

1. **金额是唯一必须的信息** — 没有金额无法进入方案计算，必须获取
2. **用途和紧急程度是可选信息** — 可以尝试询问，但用户不回答就直接继续，不要纠缠
3. **金额确认后立即推进** — 一旦金额明确，不要啰嗦，直接进入取款流程（withdraw_money 技能）

## 不问已知信息

在提问之前，必须先检查用户已经说了什么。以下信息如果用户已经提供，**绝对不要重复询问**：

| 信息项 | 用户可能的表述 | 已知则跳过 |
|-------|-------------|-----------|
| 金额 | "取5万"、"需要10万"、"3万块" | 不再问金额 |
| 用途 | "交学费"、"看病"、"急用" | 不再问用途 |
| 紧急程度 | "很急"、"不着急"、"越快越好" | 不再问时间 |

**只问用户没有提到的信息。如果金额已知，直接进入取款流程，不要多此一举。**

## 触发条件

以下情况触发本技能：
- 用户说"我想取钱"、"需要用钱"等，但**未说明金额**

以下情况**不触发**本技能：
- 用户已经明确了金额（如"我想取5万"）→ 直接进入 withdraw_money 技能
- 用户在聊保单查询、保障咨询等非取款话题

## 执行流程

### 第一步：询问金额（必须）

用户说了"想取钱"但没说金额时，简短询问：

```
请问您大概需要多少资金呢？
```

不要给选项列表，不要解释为什么需要这个信息，一句话即可。

### 第二步：可选信息（不强求）

金额确认后，如果用途和紧急程度都未知，可以**顺带一问**，但必须让用户感觉可以不答：

```
方便说一下用途和时间要求吗？这样我能推荐更合适的方案。不说也没关系，我直接帮您算。
```

**关键**：这一步是可选的。如果用户不回答或表示不想说，**立即**进入取款流程，不要追问。

### 第三步：交接给取款流程

金额明确后（无论用途/紧急程度是否已知），本技能结束，交接给 withdraw_money 技能进行方案计算。

## 注意事项

1. **最重要**：不要重复询问用户已经明确的信息
2. **不要啰嗦**：金额有了就推进，不要多余的确认和寒暄
3. **不需要保单信息**：不要问用户"从哪份保单取"，规则引擎会自动处理
4. 不要让用户觉得在被审问
5. 尊重用户隐私，用途和紧急程度是可选信息
6. 如果用户不愿回答可选信息，直接用已有信息进入方案计算
7. 不要在澄清中提及具体金额数字（如"可贷款3.3万"），金额需通过工具计算后才能给出
