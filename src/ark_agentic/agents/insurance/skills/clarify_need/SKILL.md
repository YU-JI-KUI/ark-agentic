# 需求澄清技能

当用户的需求不够明确时，使用此技能通过友好的对话收集必要信息。

## 触发条件

以下情况需要进行需求澄清：
- 用户说"我想取钱"但未说明金额
- 用户有多份保单但未指定操作哪份
- 用户需求可能有多种理解方式

## 澄清原则

### 1. 最小化提问
- 每次只问 1-2 个最关键的问题
- 避免一次性询问太多信息
- 能从上下文推断的就不要问

### 2. 提供选项
- 尽量给出选择题而非开放式问题
- 选项要覆盖常见情况
- 保留"其他"选项

### 3. 友好语气
- 使用口语化表达
- 解释为什么需要这些信息
- 表达理解和耐心

## 常见澄清场景

### 金额澄清

```
您大概需要多少资金呢？
- 1万以内
- 1-5万
- 5-10万
- 10万以上
- 我想知道最多能取多少
```

### 保单选择

```
您有以下保单可以操作：
1. 平安福终身寿险 - 可贷款约3.3万
2. 金瑞人生年金险 - 可领取约6.5万

请问您想操作哪一份？或者两份都可以？
```

### 用途了解（可选）

```
方便告诉我您的资金用途吗？这样我可以推荐更合适的方案：
- 日常开支
- 子女教育
- 医疗费用
- 其他用途（不方便说也没关系）
```

### 时间要求

```
您对资金到账时间有要求吗？
- 很急，越快越好
- 一周内到账就行
- 不着急，方便就好
```

## 注意事项

1. 不要让用户觉得在被审问
2. 解释收集信息的目的
3. 尊重用户隐私，非必要信息可跳过
4. 如果用户不愿回答，提供通用方案
