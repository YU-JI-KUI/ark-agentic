# 取款汇总卡片技能

在向用户展示取款方案前，调用此技能渲染可视化的金额汇总卡片。

## 触发条件

**同时满足以下条件时触发：**
1. 用户询问了可取金额/取款方案相关问题
2. 已通过 `rule_engine(action="list_options")` 获取到保单数据

## 调用顺序（严格执行）

```
第一步：rule_engine(action="list_options", user_id=用户ID)
         ↓ 获取保单可用金额数据
第二步：render_card(card_type="withdraw_summary", card_args=JSON字符串)
         ↓ 自动从第一步结果中提取金额，渲染可视化卡片
第三步：给出文字说明（可选，卡片本身已包含建议方案）
```

**不可颠倒顺序**：`render_card` 必须在 `rule_engine` 之后调用，因为取款汇总需要读取 rule_engine 的保单数据。

## render_card 参数说明（取款汇总）

- **card_type**：固定为 `"withdraw_summary"`。
- **card_args**：可选，JSON 对象字符串，可含以下键（均为文案，金额由工具从 rule_engine 计算）：
  - `advice_text_1`：第一条建议文案
  - `advice_text_2`：第二条建议文案
  - `plan_button_text`：按钮文案，默认"获取最优方案"
  - `plan_action_query`：点击按钮发起的问句，默认与 plan_button_text 相同

**⚠️ 不要传入金额参数**：`header_value`、`zero_cost_total`、`loan_total` 等金额字段由工具自动从 rule_engine 结果中计算，禁止在 card_args 中构造。

## 建议文案写法

### advice_text_1 示例（首选方案）
```
• 建议优先领取生存金(4,029.63元)，零成本、不影响保障、最快到账。
```

### advice_text_2 示例（补充/组合方案）
```
• 如需更多资金，可组合使用 生存金 + 保单贷款，不影响您的保障。
```

### 文案要求
- 以 `•` 开头，简洁明了
- 金额使用千分位格式（4,029.63 元）
- 不要重复卡片中已有的数据，只做总结性建议

## 完整调用示例

```python
# Step 1
rule_engine(action="list_options", user_id="U001", amount=5000)

# Step 2 — 看到 rule_engine 结果后，构造 card_args JSON 并调用通用渲染工具
render_card(
    card_type="withdraw_summary",
    card_args='{"advice_text_1":"• 建议优先领取生存金(4,029.63元)，零成本、不影响保障、最快到账。","advice_text_2":"• 如需更多资金，可组合使用 生存金 + 保单贷款，不影响您的保障。","plan_button_text":"获取最优方案","plan_action_query":"获取最优方案"}'
)
```

## 注意事项

1. 卡片中的所有金额均来自 rule_engine 计算结果，不要在文案中重新计算或编造金额
2. 卡片渲染完成后，可以给出简短的文字说明（1-2句），不要重复卡片内容
3. 卡片的"获取最优方案"按钮会触发新的对话，帮助用户进一步确认方案
