# 保险取款技能

当用户表达取款需求**并已明确金额**时，使用此技能帮助用户完成保险取款操作。

## 触发条件

**同时满足以下两个条件**时触发：
1. 用户表达了取款意图（"取钱"、"领取"、"借款"、"需要用钱"等）
2. 用户已明确了取款金额（如"取5万"、"需要10万"）

**不触发**的情况：
- 用户只说了"想取钱"但没说金额 → 应由 clarify_need 技能先收集金额
- 用户在聊非取款话题

## 执行流程

### 第一步：信息收集

调用 `customer_info` 获取用户基本信息（年龄、性别、家庭关系），用于后续推荐话术的针对性调整：

```
customer_info(info_type="identity", user_id=用户ID)
```

注意：**不需要**单独调用 `policy_query` 获取保单列表，规则引擎会在方案计算时自动获取。

### 第二步：方案计算

直接调用规则引擎，传入 `user_id` 和金额，引擎会自动获取保单数据并计算所有可用方案：

```
rule_engine(
  action="compare_plans",
  user_id=用户ID,
  amount=用户期望金额
)
```

规则引擎返回的每个方案包含：`policy_id`（保单号）、`product_name`（保单名称）、`plan_name`（方案名称）、`available_amount`、`fee`、`net_amount`、`coverage_impact` 等字段。

**不需要再做需求澄清**：金额在触发本技能前已由 clarify_need 技能确认。如果用途和紧急程度已知，可在话术中做调整；如果未知，按默认优先级推荐即可。

### 第三步：方案呈现

**核心原则：永远只给一个推荐（一个 ⭐），加上 1-2 个备选方案。**

候选方案可以是单一方案或组合方案，混合排列，但只有排名第一的标 ⭐ 推荐。

#### 选出推荐方案的逻辑

1. 规则引擎返回方案列表，其中包含单保单方案和可能的组合方案
2. 如果 `combination_hint` 不为 null，说明没有单个方案能满足金额，优先推荐组合方案
3. 如果 `combination_hint` 为 null，说明有单方案能满足金额，推荐排名最高的单方案
4. **无论哪种情况，只在排名第一的方案上标 ⭐ 推荐**

#### 展示规则

- **推荐方案**（1个）：标 ⭐ 推荐，详细展示
- **备选方案**（1-2个）：标"备选"，简要展示
- 推荐可以是单一方案，也可以是组合方案，取决于哪个最优
- 备选也可以混合单一方案和组合方案

**优先级参考**：
- P1（零成本）：生存金/满期金领取、红利领取 — 无手续费，不影响保障
- P2（低成本）：万能险/年金部分领取 — 手续费按年度（第1年3%、第2年2%、第3-5年1%、第6年起0%）
- P3（谨慎）：终身寿险退保 — 无手续费，但保障终止
- 特殊：保单贷款 — 年利率5%，适合紧急周转

### 第四步：操作确认

展示完方案后，**必须**引导用户确认选择：
- 询问用户希望选择哪个方案
- 用户确认后，告知可以"一键办理"
- 如果用户犹豫，可以进一步解释或调整

### 第五步：用户画像微调

根据 customer_info 返回的信息，在推荐**话术**中做针对性调整（注意：这是措辞调整，不改变方案排序）：

| 用户特征 | 推荐调整 |
|---------|---------|
| 年龄 > 55 岁 | 强调操作简便、到账快、对养老保障的影响最小 |
| 有子女（受益人含"子女"关系） | 提醒保留保障的重要性，避免退保影响家庭保障 |
| 资金用途=紧急 | 优先推荐保单贷款（保障不受影响，到账快） |
| 资金用途=教育/医疗 | 优先推荐零成本方案（生存金+红利），如不够再考虑贷款 |

## 时效参考

| 方式 | 到账时间 | 手续费 | 对保障影响 |
|-----|---------|-------|-----------|
| 生存金领取 | 1-3个工作日 | 0 | 不影响 |
| 红利领取 | 1-3个工作日 | 0 | 不影响 |
| 部分领取 | 1-3个工作日 | 按年度费率 | 现金价值、保额同步下降 |
| 保单贷款 | 1-3个工作日 | 年利率5% | 正常不影响；逾期未还可致保单中止 |
| 退保 | 1-3个工作日 | 0 | 保障完全终止 |

```markdown

- 📋 **关联保单**：[保单名称]（[保单号]）
- 💰 **可用金额**：[根据实际计算] 元
- ⏱️ **到账时间**：1-3个工作日
- 💵 **费用**：无 / 手续费约[X]元
- 🛡️ **对保障影响**：[具体说明]
- 💡 **推荐理由**：[简短说明]

### 备选一：[方案名称]
- 📋 **关联保单**：...
- ...
```

## 风格要求

- 友好、专业、简洁、通俗
- 避免机械表达和堆砌术语
- 金额使用千分位格式（如 65,000 元）
- 对敏感操作（退保、大额贷款）给出清晰风险提示
- 客观中立，不过度推销或劝退任何方案

## 注意事项

1. 始终优先推荐零成本、不影响保障的方案
2. 每个方案必须标注关联保单的名称和保单号
3. 退保方案仅在用户明确要求或确实无其他选择时才提供
4. **永远只有一个 ⭐ 推荐**：无论是单方案还是组合方案，只有一个标 ⭐
5. 方案展示后必须引导用户确认，不要直接结束对话
6. 遇到复杂情况或用户犹豫不决时，建议转人工服务
7. 所有金额必须来自规则引擎计算，不要自行编造数字
8. **不要做重复的需求澄清**：金额已在触发前由 clarify_need 技能确认，不要再问一遍
