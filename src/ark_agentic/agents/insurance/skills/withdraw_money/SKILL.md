# 保险取款技能

当用户表达取款、领取、借款等需求时，使用此技能帮助用户完成保险取款操作。

## 触发条件

用户表达以下意图时触发：
- "我想取点钱"
- "怎么从保单里拿钱"
- "可以领取多少"
- "保单贷款怎么办理"
- "需要用钱，怎么操作"

## 执行流程

### 1. 信息收集

首先调用工具收集必要信息：

```
1. policy_query(query_type="list") - 获取用户所有保单
2. policy_query(query_type="withdrawal_limit") - 获取可取款额度汇总
```

### 2. 需求澄清

如果用户未明确以下信息，需要主动询问：
- 期望金额范围
- 资金用途（可选，用于推荐）
- 时间要求

### 3. 方案计算

调用规则引擎计算可用方案：

```
rule_engine(action="compare_plans", policy_id=xxx, amount=xxx)
```

### 4. 方案推荐

生成 2-3 个推荐方案，每个方案包含：
- 方案名称和类型
- 可操作金额
- 费用和利息（如有）
- 对保障的影响
- 到账时间
- 推荐理由

### 5. 方案格式

使用以下格式展示方案：

```markdown
## 推荐方案

### 方案一：部分领取 ⭐推荐
- 💰 可领取：65,000元
- ⏱️ 到账：3-5个工作日
- 💡 特点：无手续费，不影响保障

### 方案二：保单贷款
- 💰 可贷款：33,600元
- 💵 年利息：约1,848元
- ⏱️ 到账：1-2个工作日
- 💡 特点：保障不变，可随时还款
```

## 注意事项

1. 始终优先推荐对客户最有利的方案
2. 明确告知每种方案的风险和影响
3. 退保方案仅在用户明确要求时提供
4. 遇到复杂情况建议转人工服务
