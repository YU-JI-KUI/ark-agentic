# 保险取款技能

当用户表达取款、领取、借款等需求时，使用此技能帮助用户完成保险取款操作。

## 触发条件

用户表达以下意图时触发：
- "我想取点钱"
- "怎么从保单里拿钱"
- "可以领取多少"
- "保单贷款怎么办理"
- "需要用钱，怎么操作"

## 执行流程

### 第一步：信息收集

调用 `customer_info` 获取用户基本信息（年龄、性别、家庭关系），用于后续推荐话术的针对性调整：

```
customer_info(info_type="identity", user_id=用户ID)
```

注意：**不需要**单独调用 `policy_query` 获取保单列表，规则引擎会在方案计算时自动获取。

### 第二步：需求澄清（仅问缺失信息）

检查用户消息中是否已包含以下信息，**只问用户尚未提供的**，已知的绝不重复问：
- 期望取款金额 → 如用户已说"取5万"，不再问金额
- 资金用途 → 如用户已说"交学费"，不再问用途
- 紧急程度 → 如用户已说"很急"，不再问时间

如果所有关键信息都已明确，直接跳到第三步方案计算。

仅在确实缺少信息时才简短询问（每次只问 1 个问题）：
- 缺金额："请问您大概需要多少资金呢？"
- 缺时间："对到账时间有要求吗？"
- 用途是可选信息，不强制询问

### 第三步：方案计算

调用规则引擎，只需传入 `user_id`，引擎会自动获取保单数据并计算所有可用方案：

```
rule_engine(
  action="compare_plans",
  user_id=用户ID,
  amount=用户期望金额
)
```

规则引擎返回的每个方案包含：`policy_id`（保单号）、`product_name`（保单名称）、`plan_name`（方案名称）、`available_amount`、`fee`、`net_amount`、`coverage_impact` 等字段。

### 第四步：方案呈现

按照规则引擎返回的**优先级顺序**向用户展示方案。每个方案必须标注关联的保单信息（保单名称 + 保单号）：

**第一优先级（零成本）**
- 生存金 / 满期金领取：无手续费，不影响保障
- 红利领取：无手续费，不影响保障

**第二优先级（低成本）**
- 万能险部分领取 / 其他部分领取：手续费较低（第1年3%、第2年2%、第3-5年1%、第6年起0%），保障有一定影响

**第三优先级（谨慎推荐）**
- 终身寿险退保：无手续费，但保障完全终止，仅在用户明确要求或无其他可行方案时才呈现

**特殊选项（按场景推荐）**
- 保单贷款：年利率5%，适合紧急周转、不愿丧失保障的情况。到账快，但需按时还款

### 第五步：操作确认

展示完方案后，**必须**引导用户确认选择：
- 询问用户希望选择哪个方案
- 用户确认后，告知可以"一键办理"
- 如果用户犹豫，可以进一步解释或调整

### 第六步：用户画像微调

根据 customer_info 返回的信息，在推荐**话术**中做针对性调整（注意：这是措辞调整，不改变方案排序）：

| 用户特征 | 推荐调整 |
|---------|---------|
| 年龄 > 55 岁 | 强调操作简便、到账快、对养老保障的影响最小 |
| 有子女（受益人含"子女"关系） | 提醒保留保障的重要性，避免退保影响家庭保障 |
| 资金用途=紧急 | 优先推荐保单贷款（保障不受影响，到账快） |
| 资金用途=教育/医疗 | 优先推荐零成本方案（生存金+红利），如不够再考虑贷款 |

## 时效参考

| 方式 | 到账时间 | 手续费 | 对保障影响 |
|-----|---------|-------|-----------|
| 生存金领取 | 1-3个工作日 | 0 | 不影响 |
| 红利领取 | 1-3个工作日 | 0 | 不影响 |
| 部分领取 | 1-3个工作日 | 按年度费率 | 现金价值、保额同步下降 |
| 保单贷款 | 1-3个工作日 | 年利率5% | 正常不影响；逾期未还可致保单中止 |
| 退保 | 1-3个工作日 | 0 | 保障完全终止 |

## 风格要求

- 友好、专业、简洁、通俗
- 避免机械表达和堆砌术语
- 金额使用千分位格式（如 65,000 元）
- 对敏感操作（退保、大额贷款）给出清晰风险提示
- 客观中立，不过度推销或劝退任何方案
- 每次只问 1-2 个问题，保持对话流畅

## 注意事项

1. 始终优先推荐零成本、不影响保障的方案
2. 每个方案必须标注关联保单的名称和保单号，方便用户对应
3. 退保方案仅在用户明确要求或确实无其他选择时才提供，并清楚说明后果
4. 当单一方案不够时，主动建议组合方案（如生存金+红利+贷款）
5. 方案展示后必须引导用户确认，不要直接结束对话
6. 遇到复杂情况或用户犹豫不决时，建议转人工服务
7. 所有金额必须来自规则引擎计算，不要自行编造数字
