# 测试技能

## 业务背景
这是一个示例测试技能，用于演示技能创建、执行和验证的完整流程。该技能模拟了一个简单的数据处理任务，可以用于测试 Agent 的技能执行能力。

## 触发条件
当用户提到以下关键词时，可以触发此技能：
- "测试技能"
- "运行测试"
- "验证功能"
- "示例任务"
- "demo"

## 执行步骤

### 1. 接收输入
- 用户提供测试数据或指定测试类型
- 如果没有指定，使用默认测试数据集

### 2. 数据处理
- 对输入数据进行标准化处理
- 验证数据格式和完整性
- 应用预设的数据转换规则

### 3. 执行测试
- 运行核心测试逻辑
- 记录执行过程中的关键指标
- 捕获任何异常或错误信息

### 4. 生成报告
- 汇总测试结果
- 提供执行统计信息
- 给出通过/失败状态
- 包含详细的时间戳和性能数据

## 输出格式

测试技能应返回以下格式的报告：

```json
{
  "status": "success|failure",
  "timestamp": "YYYY-MM-DD HH:MM:SS",
  "test_type": "用户指定的测试类型",
  "input_data": "原始输入数据摘要",
  "processed_data": "处理后数据摘要",
  "execution_time_ms": 123,
  "metrics": {
    "data_points": 100,
    "errors": 0,
    "warnings": 2
  },
  "summary": "测试执行摘要说明"
}
```

## 注意事项

1. **数据安全**: 测试数据不应包含敏感信息
2. **资源限制**: 测试执行应控制资源使用，避免影响系统性能
3. **错误处理**: 所有异常都应被捕获并记录，不应导致系统崩溃
4. **可重复性**: 相同的输入应产生相同的输出结果
5. **日志记录**: 所有测试执行都应生成详细的日志记录

## 示例对话

**用户**: 请运行测试技能
**Agent**: 正在执行测试技能... 请稍等

**用户**: 测试数据处理功能
**Agent**: 收到，开始测试数据处理功能。输入数据：示例数据集，包含100条记录。

**用户**: 验证JSON解析
**Agent**: 开始验证JSON解析功能。测试用例：解析复杂嵌套JSON结构。
