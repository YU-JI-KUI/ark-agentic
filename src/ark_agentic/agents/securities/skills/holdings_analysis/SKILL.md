# 用户资产持仓技能

## 一、技能目标

为用户提供持仓查询与分析能力，包括：

-   ETF
-   港股通
-   基金理财

支持：

1️⃣ 实时持仓获取\
2️⃣ UI卡片展示\
3️⃣ 持仓分析（盈亏、占比）

本技能仅提供客观分析，不提供投资建议。

------------------------------------------------------------------------

## 二、意图模型

```yaml
intent_schema:
  asset_type:
    enum:
      - ETF   # ETF持仓
      - HKSC  # 港股通持仓
      - FUND  # 基金/理财持仓

  mode:
    enum:
      - MODE_CARD  # 查看持仓 → 调用数据工具 + display_card 展示卡片
      - MODE_TEXT  # 分析持仓 → 调用数据工具 + 文字报告（含盈亏分析、占比分布）
```

### Mode 判断规则

**判断顺序：先看是否有 MODE_TEXT 的明确信号，没有则默认 MODE_CARD。**

#### MODE_TEXT 触发条件（必须满足至少一个）

用户表达中**显式包含**以下任意信号：

| 信号类型     | 示例关键词或句式                                          |
| ------------ | --------------------------------------------------------- |
| 盈亏询问     | 赚了、亏了、盈亏、收益、涨跌、今天涨了多少、浮盈、浮亏   |
| 比较/分析    | 分析、对比、怎么样、情况如何、表现怎样、哪个更好          |
| 占比/分布    | 占比、分布、各类、比例、多少比例                          |
| 明确要求文字 | 告诉我、说一下、帮我看看（后接分析意图）                  |

#### MODE_CARD 触发条件（默认路径）

**不含任何 MODE_TEXT 信号时，统一走 MODE_CARD。**

典型触发表达（仅作参考，非穷举）：

| 用户表达               | 判断理由                     |
| ---------------------- | ---------------------------- |
| 我的ETF                | 无分析信号，查看意图          |
| 看看港股通             | 无分析信号，查看意图          |
| 帮我看一下基金持仓     | "看一下"= 查看，非分析       |
| 我现在持有哪些基金     | 查询持仓列表，无盈亏/占比关键词 |
| 显示我的港股持仓       | 明确展示意图                  |

#### 示例对照表

| 用户表达               | asset_type | mode      |
| ---------------------- | ---------- | --------- |
| 我的ETF                | ETF        | MODE_CARD |
| 看看港股通             | HKSC       | MODE_CARD |
| 帮我查一下基金         | FUND       | MODE_CARD |
| 基金收益怎么样         | FUND       | MODE_TEXT |
| 港股通赚了多少         | HKSC       | MODE_TEXT |
| ETF今天涨跌情况        | ETF        | MODE_TEXT |

### 路由边界（不属于本 Skill）

以下问题**不由本 Skill 处理**，需路由至其他 Skill：

| 用户问题                 | 应路由至         |
| ------------------------ | ---------------- |
| 总资产/账户总览/资产概况 | `asset_overview` |
| 哪只赚最多/收益排名      | `profit_inquiry` |

## 三、工具契约

### 数据获取工具（必须先调用）

| 工具              | 用途           |
| ----------------- | -------------- |
| `etf_holdings()`  | 获取 ETF 持仓  |
| `hksc_holdings()` | 获取港股通持仓 |
| `fund_holdings()` | 获取基金持仓   |

### 展示工具（必须在数据工具之后调用）

| 工具                              | 用途                     |
| --------------------------------- | ------------------------ |
| `display_card(source_tool="xx")`  | 将已获取的数据推送至前端显示卡片 |

> ⚠️ **`display_card` 必须在对应数据工具调用成功并返回数据之后才能调用。**
> 禁止在未调用数据工具的情况下直接调用 `display_card`。

------------------------------------------------------------------------

## 四、执行流程

```
    STEP_1_INTENT_PARSE
            ↓
    STEP_2_FETCH_DATA
            ↓
        ┌───┴───┐
        ↓       ↓
  MODE_CARD  MODE_TEXT
  (展示卡片) (文字分析)
```

### STEP_1_INTENT_PARSE

解析用户意图，识别：
- `asset_type`：资产类型（ETF / HKSC / FUND）
- `mode`：MODE_CARD 或 MODE_TEXT

如果问题属于路由边界之外，**立即回复提示，不进入 STEP_2**：

> 该问题更适合通过账户总览或收益查询功能来回答，正在为您跳转…

### STEP_2_FETCH_DATA（必选）

根据 `asset_type` 调用对应工具：

| asset_type       | 调用工具                                          |
| ---------------- | ------------------------------------------------- |
| ETF              | `etf_holdings()`                                  |
| HKSC             | `hksc_holdings()`                                 |
| FUND             | `fund_holdings()`                                 |
| 未指定（占比查询）| 串行调用全部三个工具，顺序：ETF → HKSC → FUND     |

> ⚠️ **严禁并发调用多个工具**，多工具场景必须串行。
> ⚠️ **严禁从历史对话中提取数值**，必须每次实时调用工具。

### MODE_CARD（条件触发）

**触发条件**：`intent.mode == MODE_CARD`

**执行内容**：

1. 先输出一句简短确认（≤30字）
2. 调用 `display_card(source_tool="<已调用的工具名>")` 展示卡片

**失败处理**：

| 情况   | 行为                        |
| ------ | --------------------------- |
| 超时   | 不调用 display_card，文字回复 |
| 空持仓 | 不调用 display_card，文字告知 |

### MODE_TEXT（条件触发）

**触发条件**：`intent.mode == MODE_TEXT`

**执行内容**：基于工具数据输出文字分析，**禁止调用 display_card**。

分析内容按用户问题侧重选取：

| 用户侧重       | 分析内容                           |
| -------------- | ---------------------------------- |
| 盈亏/收益      | 持仓列表（名称、市值、今日盈亏）、总盈亏 |
| 分布/占比      | 各类资产市值及占总持仓百分比       |
| 综合分析       | 持仓列表 + 汇总统计 + 一句观察     |

> ⚠️ 禁止生成交易建议、买卖推荐、市场预测

------------------------------------------------------------------------

## 五、输出策略

### MODE_CARD

- 确认语 ≤30字，例如："已为您刷新并显示最新的ETF持仓信息。"
- 紧接着调用 `display_card`
- 禁止输出数值摘要或原始 JSON

### MODE_TEXT

- **总字数 ≤200字**，Markdown 格式
- 禁止调用 display_card

示例（盈亏分析）：

> 您当前持有ETF共3只，总市值约12.6万元，今日合计盈亏+320元。
>
> | 名称       | 持仓   | 今日盈亏 |
> |------------|--------|--------|
> | 沪深300ETF | 500份  | +180元 |
> | 科创50ETF  | 1000份 | +140元 |
> | 中证500ETF | 200份  | 0元    |

示例（占比分布）：

> 您当前持仓总市值约35.4万元。其中ETF持仓12.6万元（占36%）、港股通8.3万元（占23%）、基金14.5万元（占41%）。

## 六、错误处理策略

| 错误       | 回复                 |
| ---------- | -------------------- |
| 工具不可用 | 系统繁忙，请稍后重试 |
| 数据为空   | 当前无持仓           |
| 部分失败   | 已显示可获取的数据   |

-   禁止使用历史对话数据
-   每次必须实时调用工具
-   不泄露工具原始 JSON
-   不提供投资建议
