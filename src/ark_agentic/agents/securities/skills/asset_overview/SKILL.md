# 资产总览技能

## 一、技能目标

帮助用户快速了解账户的整体资产状况，包括：

-   总资产、现金、持仓市值
-   今日收益及关键指标
-   两融账户风险指标

本技能仅提供客观分析，不提供投资建议。

------------------------------------------------------------------------

## 二、意图模型

```yaml
intent_schema:
  account_type:
    enum:
      - NORMAL  # 普通账户
      - MARGIN  # 两融账户

  mode:
    enum:
      - MODE_CARD  # 查看总资产 → 调用数据工具 + display_card 展示卡片
      - MODE_TEXT  # 分析总资产 → 调用数据工具 + 文字报告（含两融风险指标）
```

### 默认推断规则

| 用户表达           | mode      |
| ------------------ | --------- |
| 查看我的资产       | MODE_CARD |
| 账户总览           | MODE_CARD |
| 我有多少钱         | MODE_CARD |
| 为什么今天亏损     | MODE_TEXT |
| 资产配置合理吗     | MODE_TEXT |
| 维持担保比率多少   | MODE_TEXT |

### 路由边界（不属于本 Skill）

以下问题**不由本 Skill 处理**，需路由至其他 Skill：

| 用户问题               | 应路由至           |
| ---------------------- | ------------------ |
| ETF/港股通/基金持仓明细 | `holdings_analysis` |
| 赚了多少/收益排名      | `profit_inquiry`   |

## 三、工具契约

### 数据获取工具（必须先调用）

| 工具                 | 用途               |
| -------------------- | ------------------ |
| `account_overview()` | 获取账户总资产数据 |

### 展示工具（必须在数据工具之后调用）

| 工具                              | 用途                         |
| --------------------------------- | ---------------------------- |
| `display_card(source_tool="xx")`  | 将已获取的数据推送至前端显示卡片 |

> ⚠️ **`display_card` 必须在 `account_overview` 调用成功并返回数据之后才能调用。**
> 禁止在未调用数据工具的情况下直接调用 `display_card`。

------------------------------------------------------------------------

## 四、执行流程

```
    STEP_1_INTENT_PARSE
            ↓
    STEP_2_FETCH_DATA
            ↓
        ┌───┴───┐
        ↓       ↓
  MODE_CARD  MODE_TEXT
  (展示卡片) (文字分析)
```

### STEP_1_INTENT_PARSE

解析用户意图，识别：
- `account_type`：账户类型（NORMAL / MARGIN），从 context 中读取
- `mode`：MODE_CARD 或 MODE_TEXT

如果问题属于路由边界之外，**立即回复提示，不进入 STEP_2**：

> 该问题更适合通过持仓查询或收益查询功能来回答，正在为您跳转…

### STEP_2_FETCH_DATA（必选）

调用 `account_overview()` 获取账户总资产数据。

> ⚠️ **严禁从历史对话中提取数值**，必须每次实时调用工具。

### MODE_CARD（条件触发）

**触发条件**：`intent.mode == MODE_CARD`

**执行内容**：

1. 先输出一句简短确认（≤30字）
2. 调用 `display_card(source_tool="account_overview")` 展示卡片

**失败处理**：

| 情况   | 行为                        |
| ------ | --------------------------- |
| 超时   | 不调用 display_card，文字回复 |
| 数据为空 | 不调用 display_card，文字告知 |

### MODE_TEXT（条件触发）

**触发条件**：`intent.mode == MODE_TEXT`

**执行内容**：基于工具数据输出文字分析，**禁止调用 display_card**。

分析内容按账户类型选取：

| 账户类型 | 分析内容                                       |
| -------- | ---------------------------------------------- |
| NORMAL   | 总资产、现金、持仓市值、今日收益               |
| MARGIN   | 以上全部 + 净资产、总负债、维持担保比率、风险等级 |

> ⚠️ 禁止生成交易建议、买卖推荐、市场预测

------------------------------------------------------------------------

## 五、输出策略

### MODE_CARD

- 确认语 ≤30字，例如："已为您刷新并显示最新的账户资产信息。"
- 紧接着调用 `display_card`
- 禁止输出数值摘要或原始 JSON

### MODE_TEXT

- **总字数 ≤200字**，Markdown 格式
- 禁止调用 display_card

示例（普通账户）：

> 您的账户总资产约68.4万元，其中现金余额21.3万元，持仓市值47.1万元。今日收益+1,240元（+0.26%）。

示例（两融账户）：

> 您的两融账户总资产约68.4万元，净资产52.1万元，总负债16.3万元。维持担保比率为318%，当前风险等级正常。今日收益+1,240元。

## 六、错误处理策略

| 错误       | 回复                 |
| ---------- | -------------------- |
| 工具不可用 | 系统繁忙，请稍后重试 |
| 数据为空   | 当前无资产数据       |
| 部分失败   | 已显示可获取的数据   |

-   每次必须实时调用工具
-   不泄露工具原始 JSON
-   不提供投资建议
