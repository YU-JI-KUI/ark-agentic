# 资产总览技能

## 目标

帮助用户快速了解账户的整体资产状况，包括总资产、现金、持仓市值、收益等关键指标。

## 触发条件

当用户询问以下内容时触发此技能：
- "查看我的资产"
- "账户总览"
- "我的资产情况"
- "总资产多少"
- 其他类似的账户整体查询

## 执行步骤（强制链路）

### 第一步 — 数据拉取（必选）

调用 `account_overview()` 工具获取账户总资产数据。

> ⚠️ **严禁从历史对话中提取数值**，必须每次重新调用工具。

### 第二步 — 视觉呈现（必选）

获取数据后，**必须立即调用 `display_card(source_tool="account_overview")`** 将数据推送至前端界面显示为卡片。

### 第三步 — 文字回复（分流）

根据用户意图，仅执行以下一种：

- **意图 A（仅查看）**：如"查看资产"、"账户总览" → 简短回复："已为您刷新并显示最新的账户资产卡片。"（**禁止输出任何数据汇总或 JSON**）
- **意图 B（要分析）**：如"为什么亏损" → 根据工具返回数据用 **Markdown 格式撰写分析**

判断账户类型：
- 普通账户：展示基本资产信息
- 两融账户：额外展示维持担保比率、风险等级等

## 输出格式

### 场景 1：纯总览查询（简短确认）

**触发条件**：
- 用户输入简单，如"查看资产"、"账户总览"
- 没有额外的分析性问题

**你的回复**：
简短确认即可，例如："已为您刷新并显示最新的账户资产卡片。"

> 不要在回复文本中输出 JSON 数据或工具返回的原始内容。

### 场景 2：分析性查询（Markdown 分析）

**触发条件**：
- 用户询问具体问题，如"为什么今天亏损"、"资产配置合理吗"

**你的回复**：
基于工具返回的数据，用 Markdown 格式撰写分析报告：

```markdown
## 您的资产总览

- 💰 **总资产**：¥1,250,000.00
- 💵 **可用资金**：¥50,000.00
- 📈 **持仓市值**：¥1,200,000.00
- 📊 **今日收益**：+¥15,000.00 (+1.2%)
- 🎯 **累计收益**：+¥250,000.00 (+25.0%)

### 分析

今日收益为正，主要得益于...
```

## 两融账户特殊处理

如果是两融账户，需要额外展示：

```markdown
**两融账户风险指标**：
- 🛡️ **维持担保比率**：280%
- ⚠️ **风险等级**：低风险
- 💰 **维持保证金**：¥850,000
- 💵 **可用保证金**：¥1,650,000
```

## 注意事项

1. **数据时效性**：提醒用户数据更新时间
2. **风险提示**：两融账户需要特别提示风险等级
3. **简洁性**：总览应简洁明了，避免过多细节
4. **合规要求**：禁止给出"建议买入/卖出/持有"等具体投资建议，只客观呈现数据和事实分析
