# 资产总览技能

## 一、技能目标

帮助用户快速了解账户的整体资产状况，包括：

- 总资产
- 现金
- 持仓市值
- 收益等关键指标

支持：

1️⃣ 实时资产获取\
2️⃣ UI卡片展示\
3️⃣ 数据分析\
4️⃣ 两融账户风险指标

本技能仅提供客观分析，不提供投资建议。

------------------------------------------------------------------------

## 二、用户意图识别（用于 Router/Planner）

```yaml
intent_schema:
  account_type:
    enum:
      - NORMAL   # 普通账户
      - MARGIN   # 两融账户

  action:
    enum:
      - VIEW     # 仅查看，展示卡片
      - ANALYZE  # 分析，输出 Markdown 报告
```

### 默认推断规则

| 用户表达       | action  |
| -------------- | ------- |
| 查看我的资产   | VIEW    |
| 账户总览       | VIEW    |
| 为什么亏损     | ANALYZE |
| 资产配置合理吗 | ANALYZE |

## 三、工具契约

### 数据获取工具

| 工具                 | 用途                 |
| -------------------- | -------------------- |
| `account_overview()` | 获取账户总资产数据   |

### 展示工具

| 工具                               | 用途                   |
| ---------------------------------- | ---------------------- |
| `display_card(source_tool="xxx")`  | 将数据推送至前端显示卡片 |

------------------------------------------------------------------------

## 四、执行状态机

```
    STATE_1_INTENT_PARSE
            ↓
    STATE_2_FETCH_DATA
            ↓
    ┌───────┴───────┐
    ↓               ↓
STATE_3_ANALYSIS   STATE_4_CARD_DISPLAY
(intent.action == ANALYZE)  (intent.action == VIEW)
```

### STATE_1_INTENT_PARSE

解析用户意图，识别：
- `account_type`：账户类型（NORMAL / MARGIN）
- `action`：查看或是需要分析（VIEW / ANALYZE）

### STATE_2_FETCH_DATA（必选）

调用 `account_overview()` 工具获取账户总资产数据。

> ⚠️ **严禁从历史对话中提取数值**，必须每次重新调用工具。

### STATE_3_ANALYSIS（条件触发）

**触发条件**：`intent.action == ANALYZE`

**执行内容**：输出 Markdown 分析报告

分析模块包含：

#### 基础统计

-   总资产、现金、持仓市值
-   今日收益、累计收益

#### 两融账户风险指标

-   维持担保比率
-   风险等级
-   维持保证金、可用保证金

#### 风险提示

-   两融账户风险等级提示
-   大额亏损标记

> ⚠️ 禁止生成交易建议

### STATE_4_CARD_DISPLAY（条件触发）

**触发条件**：`intent.action == VIEW`

**执行内容**：

1. 简短确认回复
2. 调用 `display_card(source_tool="account_overview")` 展示卡片

**失败处理**：

| 情况       | 行为               |
| ---------- | ------------------ |
| 超时       | 不调用 display_card |
| 数据为空   | 提示无数据         |
| 部分成功   | 显示成功部分       |

------------------------------------------------------------------------

## 五、输出策略

### 查询类（action == VIEW）

简短确认回复，例如："已为您刷新并显示最新的账户资产信息。"

> 禁止输出数值摘要或原始 JSON

### 分析类（action == ANALYZE）

根据工具返回的数据，用 Markdown 格式撰写分析报告，建议包含：

- **资产概览**：总资产、可用资金、持仓市值、收益等关键指标
- **两融指标**：维持担保比率、风险等级等（如适用）
- **分析观察**：基于数据的客观描述

### 两融账户特殊处理

如果账户类型为两融账户（MARGIN），需要额外展示风险指标：
- 维持担保比率
- 风险等级
- 维持保证金、可用保证金

------------------------------------------------------------------------

## 六、错误处理策略

| 错误         | 回复                   |
| ------------ | ---------------------- |
| 工具不可用   | 系统繁忙，请稍后重试   |
| 数据为空     | 当前无资产数据         |
| 部分失败     | 已显示可获取的数据     |

------------------------------------------------------------------------

## 七、性能与安全约束

-   禁止使用历史对话数据
-   每次必须实时调用工具
-   不泄露工具原始 JSON
-   不提供投资建议
