# 收益查询技能

## 目标

帮助用户了解投资收益情况，包括今日收益、累计收益、各类资产收益对比等。

## 触发条件

当用户询问以下内容时触发此技能：
- "今天赚了多少"
- "总收益多少"
- "哪个资产收益最好"
- "收益率排名"
- 其他类似的收益查询

## 执行步骤（强制链路）

### 第一步 — 数据拉取（必选）

确定查询范围，调用对应的数据工具获取最新实时数据：

- 总收益 → 调用 `account_overview()`
- 单类资产收益 → 调用对应持仓工具（如 `etf_holdings()`）
- 具体标的收益 → 调用 `security_detail(security_code=...)`

> ⚠️ **严禁从历史对话中提取数值**，必须每次重新调用工具。

### 第二步 — 视觉呈现（必选）

获取数据后，**必须立即调用 `display_card(source_tool="xxx")`** 将数据推送至前端界面显示为卡片。

例如：查询总收益后，立即调用 `display_card(source_tool="account_overview")`。

### 第三步 — 文字回复（分流）

根据用户意图，仅执行以下一种：

- **意图 A（仅查看）**：如"今天赚了多少"、"总收益" → 简短回复："已为您刷新并显示最新的收益卡片。"（**禁止输出任何数据汇总或 JSON**）
- **意图 B（要分析）**：如"为什么今天亏损" → 根据工具返回数据用 **Markdown 格式撰写分析**。分析时：
  - 识别收益来源
  - 对比市场表现

## 输出格式

### 场景 1：简单收益查询（简短确认）

**触发条件**：
- "今天赚了多少"、"总收益"等简单查询

**你的回复**：
简短确认即可，例如："已为您刷新并显示最新的收益卡片。"

> 不要在回复文本中输出 JSON 数据或工具返回的原始内容。

### 场景 2：收益分析（Markdown 分析）

**触发条件**：
- "为什么今天亏损"、"收益来源分析"

**你的回复**：
基于工具返回的数据，用 Markdown 格式撰写分析报告：

```markdown
## 收益分析

### 今日收益

- 📊 **今日盈亏**：+¥15,000 (+1.2%)
- 🎯 **累计收益**：+¥250,000 (+25.0%)

### 收益来源

| 资产类别 | 今日收益 | 贡献度 |
|---------|---------|--------|
| ETF | +¥300 | 2% |
| 港股通 | +¥2,500 | 16.7% |
| 基金理财 | +¥50 | 0.3% |
| 其他 | +¥12,150 | 81% |

### 分析

今日收益主要来自港股通板块，腾讯控股上涨带动...
```

## 收益排名

展示各持仓的收益率排名：

```markdown
## 收益率排名 TOP 3

1. 🥇 **创业板ETF**：+6.82%
2. 🥈 **沪深300ETF**：+6.67%
3. 🥉 **招商白酒基金**：+12.5%
```

## 亏损提示

如果有亏损，需要特别提示：

```markdown
⚠️ **亏损提示**

以下持仓当前处于亏损状态：
- 某某股票：-5.2%
```

## 注意事项

1. **数据准确性**：确保收益计算准确
2. **时间范围**：明确收益的时间范围（今日/累计）
3. **风险提示**：过往收益不代表未来表现
4. **合规要求**：禁止给出"建议买入/卖出/持有"等具体投资建议，只客观呈现数据和事实分析
