# 收益查询技能

## 一、技能目标

帮助用户了解投资收益情况，包括：

- 今日收益
- 累计收益
- 各类资产收益对比
- 收益率排名

支持：

1️⃣ 实时收益获取\
2️⃣ UI卡片展示\
3️⃣ 收益分析\
4️⃣ 亏损提示

本技能仅提供客观分析，不提供投资建议。

------------------------------------------------------------------------

## 二、意图模型（用于 Router/Planner）

```yaml
intent_schema:
  scope:
    enum:
      - TOTAL          # 总收益
      - ASSET_TYPE     # 单类资产收益
      - SECURITY       # 具体标的收益

  action:
    enum:
      - VIEW           # 仅查看，展示卡片
      - ANALYZE        # 分析，输出 Markdown 报告
      - RANKING        # 收益排名
```

### 默认推断规则

| 用户表达         | action  | scope      |
| ---------------- | ------- | ---------- |
| 今天赚了多少     | VIEW    | TOTAL      |
| 总收益多少       | VIEW    | TOTAL      |
| 为什么今天亏损   | ANALYZE | TOTAL      |
| ETF收益怎么样    | ANALYZE | ASSET_TYPE |
| 哪个资产收益最好 | RANKING | TOTAL      |

------------------------------------------------------------------------

## 三、工具契约

### 数据获取工具

| 工具                          | 用途                     |
| ----------------------------- | ------------------------ |
| `account_overview()`          | 获取总收益数据           |
| `etf_holdings()`              | 获取 ETF 持仓及收益      |
| `hksc_holdings()`             | 获取港股通持仓及收益     |
| `fund_holdings()`             | 获取基金持仓及收益       |
| `security_detail(security_code=...)` | 获取具体标的收益 |

### 展示工具

| 工具                               | 用途                   |
| ---------------------------------- | ---------------------- |
| `display_card(source_tool="xxx")`  | 将数据推送至前端显示卡片 |

------------------------------------------------------------------------

## 四、执行状态机

```
    STATE_1_INTENT_PARSE
            ↓
    STATE_2_FETCH_DATA
            ↓
    ┌───────┴───────┐
    ↓               ↓
STATE_3_ANALYSIS   STATE_4_CARD_DISPLAY
(intent.action != VIEW)  (intent.action == VIEW)
```

### STATE_1_INTENT_PARSE

解析用户意图，识别：
- `scope`：查询范围（TOTAL / ASSET_TYPE / SECURITY）
- `action`：操作类型（VIEW / ANALYZE / RANKING）

### STATE_2_FETCH_DATA（必选）

根据 `scope` 调用对应工具：

| scope        | 调用工具                                  |
| ------------ | ----------------------------------------- |
| TOTAL        | `account_overview()`                      |
| ASSET_TYPE   | 对应持仓工具（如 `etf_holdings()`）       |
| SECURITY     | `security_detail(security_code=...)`      |

> ⚠️ **严禁从历史对话中提取数值**，必须每次重新调用工具。

### STATE_3_ANALYSIS（条件触发）

**触发条件**：`intent.action != VIEW`

**执行内容**：输出 Markdown 分析报告

分析模块包含：

#### 收益统计

-   今日盈亏、累计收益
-   收益率

#### 收益来源分析

-   各资产类别收益贡献
-   收益来源识别

#### 收益排名

-   收益率 TOP 排名
-   亏损持仓提示

> ⚠️ 禁止生成交易建议

### STATE_4_CARD_DISPLAY（条件触发）

**触发条件**：`intent.action == VIEW`

**执行内容**：

1. 调用 `display_card(source_tool="xxx")` 展示卡片
2. 简短确认回复

**失败处理**：

| 情况       | 行为               |
| ---------- | ------------------ |
| 超时       | 不调用 display_card |
| 数据为空   | 提示无数据         |
| 部分成功   | 显示成功部分       |

------------------------------------------------------------------------

## 五、输出策略

### 查询类（action == VIEW）

简短确认回复，例如："已为您刷新并显示最新的收益信息。"

> 禁止输出数值摘要或原始 JSON

### 分析类（action != VIEW）

根据工具返回的数据，用 Markdown 格式撰写分析报告，建议包含：

- **收益概览**：今日盈亏、累计收益、收益率等关键指标
- **收益来源**：各资产类别的收益贡献
- **分析观察**：基于数据的客观描述

### 收益排名（RANKING）

展示各持仓的收益率排名，可包含 TOP 表现和亏损提示。

------------------------------------------------------------------------

## 六、错误处理策略

| 错误         | 回复                   |
| ------------ | ---------------------- |
| 工具不可用   | 系统繁忙，请稍后重试   |
| 数据为空     | 当前无收益数据         |
| 部分失败     | 已显示可获取的数据     |

------------------------------------------------------------------------

## 七、性能与安全约束

-   禁止使用历史对话数据
-   每次必须实时调用工具
-   不泄露工具原始 JSON
-   不提供投资建议
